Bot tag = @rcn_check_bot

Функционал бота: 
1) Пинг сайта по адресу, отправленному пользователем -> код состояния HTTP + само состояние;
   Запрос к сайту осуществляется с российского прокси, г. Калининград

2) Проверка нахождения введённого домена в списке заблокированных сайтов РКН (81137 шт. на момент 07.04.2026)

3) Новостной парсер на NewsAPI выводит новости о возможной блокировке домена.

                                                            РАЗРАБОТКА
                                                            
Изначальную концепцию бота с анализом новостей в ТГ-КАНАЛАХ, а не в новостных сайтах, пришлось свернуть:
Регистрация тг-приложений и юзерботов, необходимых для работы с каналами, сейчас практически невозможна из-за проблем с смс-кодами
<img width="743" height="667" alt="image" src="https://github.com/user-attachments/assets/d4033788-eeeb-4c48-bfc8-c8269b7b7ebc" />
Использование виртуальной сети в любом виде также не даёт работать с api; наличие телефонного номера в другой стране не помогает.


Таким образом, новости берутся из ограниченного пула источников NewsAPI; из них простым фильтром почти всегда вычленяются новости с
однозначным ответом на вопрос о блокировке, поэтому анализировать их с помощью ИИ не имеет смысла.

Для того, чтобы пинг сайта и вывод его html состояния происходил из РФ, мы также подняли прокси сервер на Ubuntu, задействованный в site_checker.py
Список заблокированных доменов взять xml парсингом с официального сайта РКН.

Итого, наш бот одновременно даёт пользователю конкретный ответ на вопрос: упал ли сайт или его работа искуственно ограничена в Российской федерации +
выдаёт мнение основных российских новостных агрегаторов по этому вопросу.

Сайты с похожим функционалом заполнены рекламой и "ожиданиями" результата пользовательских запросов, занимающими по нескольку минут;
Доступа к БД РКН они вообще могут не иметь или использовать БД, не отражающую реальное положение вещей
